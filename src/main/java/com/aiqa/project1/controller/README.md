# controller

本目录存放 REST 控制器，负责暴露用户、标签、文档、问答和 SSE 接口。

## 主要文件

- `UserController`: 用户注册、批量注册、登录、个人信息查询、邮箱/密码/手机号修改、用户删除和仪表盘数据接口。
- `SpecialTagController`: 标签创建、批量创建、加入标签、标签列表、标签用户列表、删除标签和移除用户等接口。
- `DocsController`: 文档上传、批量上传、详情查询、版本查询、列表查询、下载和删除接口。
- `AIAnswerController`: 问答请求提交、回答状态查询、会话创建、会话列表、聊天记录、重命名、删除和收藏接口。
- `SseController`: 建立 SSE 长连接，用于向前端推送文档处理等异步状态。

## 分层职责

控制器只应处理 HTTP 参数、鉴权信息、结果包装和异常转换。业务规则应下沉到 `service/impl`，数据访问应下沉到 `mapper` 或专用工具组件。

## 维护提示

- 新增接口时保持统一返回 `Result`。
- 需要用户身份的接口应通过 `JwtUtils.validateJwt` 或拦截器传递的 `AuthInfo` 读取身份信息。
- 文件上传接口需要关注文件为空、后缀、大小、异步状态回写和异常回滚。
