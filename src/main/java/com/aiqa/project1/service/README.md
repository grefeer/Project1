# service

本目录存放业务服务接口，定义控制器可以调用的业务能力。

## 主要文件

- `UserService`: 用户注册、批量注册、登录、信息查询、账号信息修改、删除用户和仪表盘数据。
- `DocsService`: 单文件上传、批量上传、文档详情、文档删除、下载和分页列表查询。

## 分层职责

接口用于隔离控制器和实现类，让控制器只依赖业务能力而不是具体数据库或消息队列细节。实际实现位于 `service/impl`。

## 维护提示

- 新增控制器业务能力时，优先在接口中定义清晰方法，再在 `impl` 中实现。
- 接口参数应表达业务语义，避免直接泄漏过多底层实现细节。
- 抛出的业务异常应统一使用 `BusinessException` 或统一返回 `Result`。
