# service.impl

本目录存放业务服务实现，是控制器、数据库、缓存、对象存储、消息队列和 RAG 图之间的主要编排层。

## 主要文件

- `UserServiceimpl`: 用户注册、批量导入、登录、信息修改、删除用户、仪表盘统计和新用户标签初始化。
- `DocsServiceimpl`: 文档上传、版本管理、权限校验、COS 路径生成、文档删除、下载、列表查询和文档处理消息发送。
- `SpecialTagService`: 标签创建、批量导入、加入标签、标签/用户列表查询、标签删除和用户移除。
- `QuestionAnsweringService`: 问答请求登记、RAG 模式选择、回答状态查询、会话列表、聊天记录、会话创建/重命名/删除和收藏。
- `RabbitProducerService`: RabbitMQ 消息发送封装。

## 依赖关系

实现类通常依赖 `mapper` 完成数据库操作，依赖 `utils` 处理 JWT、Redis、Milvus、对象存储、SSE 和缓存，必要时通过 RabbitMQ 将耗时任务交给 `consumer`。

## 维护提示

- 这里是事务边界最集中的目录，新增数据库和外部存储组合操作时要明确回滚策略。
- 不同服务对用户角色、标签权限和文档状态有共同假设，改动时要全局搜索相关字段。
- 异步任务发出前应确保必要数据库记录和 Redis 状态已经准备好。
