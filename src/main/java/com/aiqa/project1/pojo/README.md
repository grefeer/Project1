# pojo

本目录存放项目通用数据对象，包括统一返回体、鉴权信息、统计视图、死信实体，以及按领域拆分的子包。

## 主要文件

- `Result`: 统一接口返回对象。
- `ResponseCode`: 响应码枚举。
- `AuthInfo`: JWT 解析后的用户身份信息。
- `DailyCountVO`: 按日期聚合统计的返回对象。
- `DeadLetter`: RabbitMQ 死信消息持久化实体。

## 子目录

- `document`: 文档实体、版本实体、上传/查询/列表 DTO。
- `user`: 用户实体和登录、注册、信息展示、批量导入 DTO。
- `tag`: 组织标签、用户标签、标签统计和列表 DTO。
- `qa`: 会话、聊天记录、检索决策等问答相关对象。
- `nodes`: RAG 图、状态对象和工具元数据。
- `graph`: 简单图状态对象。

## 维护提示

- Entity 类要保持与数据库表结构一致。
- DTO 类应按接口语义命名，避免直接把数据库实体暴露给前端。
- Lombok 注解变更会影响序列化、MyBatis 映射和测试构造方式。
